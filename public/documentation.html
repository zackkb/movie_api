My Movies App

Web application will provide users with information about movies, directors, and genres.
Users will be able to sign up, create a favorites list, and update personal information.



URL requests and endpoints


Client Request URL HTTP Method Request Body Data Format Response Body Data Format
Return a list of ALL movies to the user /movies GET none A JSON object holding data about all the movies
Return data (description, genre, director, image URL, whether it’s featured or not) about a single movie by title to the user /movies/:title GET none A JSON object holding data about a single movie. Example:
                {
                  "Genre": {
                      "Name": "Animated",
                      "Description": "Animated film is a collection of illustrations that are photographed frame-by-frame and then played in a quick succession."
                  },
                  "Director": {
                      "Name": "Hiroyuki Okiura",
                      "Bio": "Hiroyuki Okiura is a Japanese animation director and animator working for Production I.G.",
                      "Birthday": "1966"
                  }
                }
Return data about a genre (description) by name/title (e.g., “Comedy”) /movies/genres/:genre_name GET none A JSON object holding data about a genre. Example:
                        {
                          "Name": "Comedy",
                          "Description": "Comedy is a category of film which emphasizes humor. These films are designed to make the audience laugh through amusement."
                        }
Return data about a director (bio, birth year, death year) by name /movies/directors/:director_name GET none A JSON object holding data about a director. Example:
                    {
                      "Name": "Hiroyuki Okiura",
                      "Bio": "Hiroyuki Okiura is a Japanese animation director and animator working for Production I.G.",
                      "Birthday": "1966"
                    }
Allow new users to register /users POST A JSON object holding the user information. Example:
                        {
                          "Username": "Zach",
                          "Email": "zackkb@gmail.com",
                          "Password" "passw0rd123",
                          "Birthday": "05.21.1988"
                        }
A JSON object holding the newly created user, including ObjectId. Example:
                        {
                          "Username": "Zach",
                          "Email": "zackkb@gmail.com",
                          "Password": "passw0rd123",
                          "Birthday": "05.21.1988",
                          "FavoriteMovies": ['A Silent Voice', 'Tokyo Godfathers']
                        }
Return a list of all users /users GET none A JSON object holding data about all users
Return data on a single user by name /users/:user_name GET none A JSON object holding data about the requested user
Allow users to update their user info /users/:user_name PUT A JSON object holding the information that should be updated. Example:
          {
            "Password": "passw0rd123"
          }
A JSON object holding data about the updated user
Allow users to add a movie to their list of favorites /users/:user_name/movies/:title POST none A JSON object holding data about the updated user
Allow users to remove a movie from their list of favorites /users/:user_name/movies/:title DELETE none A JSON object holding data about the updated user
Allow existing users to deregister /users/:user_name DELETE none A success message indicating that the profile was successfully deleted.
